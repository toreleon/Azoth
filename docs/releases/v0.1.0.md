# Azoth v0.1.0

Release date: 2026-05-04

Azoth v0.1.0 is the first consolidated public release of the terminal-native
Vietnam equity research and trading copilot. It packages the full Azoth
workflow: chat-first research, multi-agent investment analysis, market-data
tools, paper trading, backtesting, risk controls, and optional
DNSE Entrade X live broker support.

## Highlights

- Interactive Ink terminal UI with streaming chat, slash commands, tool chips,
  status bar, local cards, session history, and first-time LLM setup.
- Claude Agent SDK orchestration with constrained Azoth tool access and
  resumable local project sessions.
- Multi-agent research desk with technical, fundamentals, news, sentiment, bull,
  bear, research manager, trader, risk, and portfolio roles.
- Vietnam market tooling for quotes, OHLCV, technical indicators, fundamentals,
  CafeF news, macro indices, foreign flow, ticker discovery, and portfolio state.
- Chat-first workflows for `/team`, `/analyze`, `/quote`, `/positions`,
  `/backtest`, `/health`, and `/autonomy`.
- Broker-aware execution with advisory, confirm, and auto autonomy modes.
- Paper broker with persistent SQLite cash, positions, orders, fills, and
  rejected-order records.
- Optional DNSE Entrade X / LightSpeed live broker adapter with read-only live
  contract tests and explicit arming requirements.
- Risk guardrails for lot sizing, order notional, concentration, market session,
  whitelist, no-margin cash checks, daily loss halt, and drawdown buy freeze.
- Team-driven backtesting with a built-in replay strategy, paper fills, fees,
  guardrail rejections, benchmark comparison, and max-drawdown accounting.
- Local-first runtime state under `~/.azoth` for config, SQLite database,
  market cache, broker records, team runs, and session logs.
- Public npm package support for `npx @toreleon/azoth` and global CLI installs.

## Package Command

The npm package exposes one public command:

- `azoth`

## Release And Packaging

- Published as the public scoped package `@toreleon/azoth`.
- Includes MIT license, repository metadata, release documentation, and one
  packaged CLI entry point.
- Uses Changesets and GitHub Actions for version PRs, validation, npm publish,
  and GitHub release creation.
- Allows native dependency install scripts required by `better-sqlite3` in CI.
- Reads the displayed TUI version from the installed package metadata instead
  of a hard-coded string.

## Validation

Release checks for this baseline:

- `pnpm typecheck`
- `pnpm test`
- `pnpm build`
- `npm pack --dry-run`

No release validation places live broker orders. DNSE live checks remain
read-only unless live trading is explicitly configured and armed by the user.
